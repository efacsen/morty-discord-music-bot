<task type="auto">
  <name>Task 2: Create tsconfig.json and src/types/index.ts</name>
  <files>tsconfig.json, src/types/index.ts</files>
  <action>
1. Create `tsconfig.json` in project root with these exact settings (per user decisions):
   ```json
   {
     "compilerOptions": {
       "target": "ES2022",
       "module": "NodeNext",
       "moduleResolution": "NodeNext",
       "outDir": "dist",
       "rootDir": "src",
       "strict": true,
       "allowJs": true,
       "checkJs": false,
       "esModuleInterop": true,
       "skipLibCheck": true,
       "sourceMap": true,
       "declaration": true,
       "declarationMap": true
     },
     "include": ["src/**/*"],
     "exclude": ["node_modules", "dist"]
   }
   ```
   - `strict: true` only -- no additional strict flags per user decision
   - `module: "NodeNext"` per user decision (not "node16")
   - `allowJs: true, checkJs: false` -- lets TS compile mixed codebase without erroring on .js files
   - `sourceMap: true` for debugging; `declaration`/`declarationMap` for future-proofing (Claude's discretion)

2. Create `src/types/index.ts` with the shared type layer:
   - `CommandModule` interface: `data` (SlashCommandBuilder | Omit<SlashCommandBuilder, 'addSubcommand' | 'addSubcommandGroup'>) + `execute` (ChatInputCommandInteraction => Promise<void>)
   - `EventModule` interface: `name` (string), optional `once` (boolean), `execute` ((...args: unknown[]) => Promise<void> | void)
   - `QueueMetadata` interface: `channel` (TextBasedChannel) + `requestedBy` (User)
   - Module augmentation: `declare module 'discord.js'` adding `commands: Collection<string, CommandModule>` to `Client`
   - Import types from discord.js: SlashCommandBuilder, ChatInputCommandInteraction, TextBasedChannel, User, Collection
   - Use `import type` syntax for all discord.js imports (type-only imports)

   Use the exact code from RESEARCH.md Pattern 4 as the reference implementation.

3. Run `npm run build` to verify compilation succeeds. The build will compile both .js and .ts files in src/ to dist/. Expect zero errors.
